## IMPORTANT: String Input Handling

**Critical**: C library functions expect null-terminated strings, but fuzzer's raw input data is NOT null-terminated. Always ensure string parameters (char*, const char*) are properly null-terminated to avoid false-positive crashes that aren't real bugs.

## Best Practices for C Fuzzing

1. **Memory Management**: Always check malloc/calloc return values and free allocated memory
2. **Bounds Checking**: Verify array indices and buffer sizes before access
3. **Null Termination**: Ensure strings are properly null-terminated
4. **Size Validation**: Check that size parameter is sufficient for operations
5. **Error Handling**: Handle all possible error return values from library functions
